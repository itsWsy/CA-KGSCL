==================================================
二、最高优先级：预计算并缓存 item-level KG reliability
==================================================

当前公式必须保持不变：

Substitute reliability：
K_sub(i) = max 或 mean { Cor(i, j) | j 属于 substitution neighbors of i }

Insert reliability：
K_ins(i) = max 或 mean { Cor(i, j) | j 属于 complementary neighbors of i }

问题：
这些 reliability 只由 item id、relation set、correlation score 和 agg 方式决定，与 batch、epoch 和当前 hidden state 无关，因此不能在每个 batch 中重复扫描知识图谱。

请实现一次性预计算：

sub_reliability_table[item_id]
ins_reliability_table[item_id]

要求：

1. 在数据加载完成、模型初始化或训练开始前只计算一次；
2. 表长度必须与 item embedding 的行数一致；
3. padding id 的 reliability 为 0；
4. 非法 item id 不写入；
5. relation set 中若包含 item 自身，沿用当前模块的原有过滤规则；
6. agg=max 时结果必须与原实现一致；
7. agg=mean 时结果必须与原实现一致；
8. correlation 缺失时沿用当前代码的默认值；
9. 不改变 normalization 规则；
10. 预计算结果保存为 torch.float32 Tensor；
11. 使用 register_buffer 注册到模型中，建议 persistent=False；
12. buffer 自动跟随 model.to(device) 移动到 GPU；
13. 不作为可训练参数；
14. 不参与梯度计算；
15. 避免每个 batch 从 CPU 重复拷贝到 GPU。

建议结构：

self.register_buffer(
    "sub_reliability_table",
    precomputed_sub_reliability_tensor,
    persistent=False
)

self.register_buffer(
    "ins_reliability_table",
    precomputed_ins_reliability_tensor,
    persistent=False
)

训练时必须直接通过索引取得：

sub_reliability = self.sub_reliability_table[item_seq]
ins_reliability = self.ins_reliability_table[item_seq]

其中：
- item_seq shape 为 [B, L]；
- 输出 shape 为 [B, L]；
- 不允许再次遍历 relation set；
- 不允许再次调用 get_relation_neighbors；
- 不允许在 batch 内再次扫描 correlation score。

如果当前 normalization 是“每个样本在有效位置上做 min-max normalization”，仍然保留该步骤，但只对索引得到的 [B, L] reliability 做张量归一化。

如果当前代码已经存在 reliability table：
1. 检查是否真的只构建一次；
2. 检查是否每个 batch 又重新转换成 tensor；
3. 检查是否仍在 CPU；
4. 检查是否每个 batch重复做 Python 字典查询；
5. 修复这些问题。

==================================================
三、向量化 position importance，移除逐样本位置循环
==================================================

原公式保持不变：

I_i = similarity(h_i, h_u)

其中：
- h_i 为每个有效位置的 hidden state；
- h_u 为最后一个有效位置的 hidden state；
- similarity 保持 cosine 或 dot；
- 每个样本在有效位置上做 min-max normalization；
- padding 位置为 0；
- 相同值和长度为 1 的特殊处理保持当前功能不变。

必须实现纯张量计算，避免：

for b in range(batch_size):
    values = ...
    min(...)
    max(...)

建议向量化实现：

1. seq_mask shape [B, L]；
2. 使用 position_ids 和 masked max 获取最后一个有效位置；
3. 通过 gather 获取 user_repr；
4. 一次性计算 [B, L] raw importance；
5. masked_min：将 padding 位置填为 +inf，再沿 dim=1 取 min；
6. masked_max：将 padding 位置填为 -inf，再沿 dim=1 取 max；
7. denominator = max - min；
8. 正常样本：(raw - min) / (denominator + eps)；
9. denominator 过小的样本：按当前逻辑，将最后一个有效位置设置为 1，其余有效位置设置为 0；
10. padding 位置最终设置为 0。

禁止：
- 对 batch 逐样本循环；
- 对位置逐元素循环；
- 每个 batch 反复创建 position_ids；
- 频繁调用 .item()。

可以缓存 position_ids：
如果 max_len 固定，可注册或缓存 [1, L] 的 position_ids，并按 batch expand。

==================================================
四、向量化 reliability normalization
==================================================

从缓存表索引后得到：

sub_reliability_raw = sub_table[item_seq]
ins_reliability_raw = ins_table[item_seq]

请保留原有“每个样本、有效位置范围内 min-max normalization”的功能，但改为向量化 masked min-max。

要求：
1. padding 为 0；
2. 全 0 样本保持全 0；
3. 不产生 NaN；
4. 不使用 Python batch 循环；
5. 不改变 reliability 数值语义。

建议实现通用函数：

masked_minmax_normalize(values, mask, zero_when_constant=True, eps=1e-8)

用于：
- position importance；
- substitute reliability；
- insert reliability；
- 如果当前实现还对 position scores 再归一化，也可复用。

==================================================
五、位置评分必须保持纯张量操作
==================================================

公式保持不变：

sub_scores = sub_reliability * (1.0 - importance)
ins_scores = ins_reliability * importance

要求：
1. 一次性计算 [B, L]；
2. padding mask 后设为 0；
3. 不使用 Python 循环；
4. importance 和 reliability 使用 detach；
5. 位置选择前不执行 .cpu().tolist()；
6. 不创建多余副本；
7. 在 torch.no_grad() 中执行。

==================================================
六、优化位置选择，但不得改变采样语义
==================================================

必须保留：
- position_select_mode=sample 时仍然是按 softmax 概率无放回采样；
- position_select_mode=topk 时仍然是 Top-K；
- k = ceil(ratio * valid_length)；
- fallback 逻辑不变；
- 非零位置不足 k 时的补齐逻辑不变；
- 不选择 padding；
- 不重复选择同一位置。

优先级如下：

第一优先级：
先保留当前位置选择算法，只移除 relation/correlation 的逐位置查询。
如果运行时间已经接近原始 KGSCL，则不要继续做高风险重构。

第二优先级：
如果 position selection 仍是明显瓶颈，可按相同 k 对 batch 行分组。

原因：
torch.multinomial 对二维概率矩阵可一次处理同样 num_samples 的多行。

实现建议：
1. 计算每个样本 k；
2. 按 k 的不同取值分组；
3. 对同一个 k 的样本组成概率矩阵；
4. 一次调用 torch.multinomial(prob_matrix, k, replacement=False)；
5. 再映射回原 batch 行；
6. fallback 样本单独处理；
7. 保持每个样本的概率分布和无放回语义不变。

注意：
- 不要求与旧实现逐位完全相同的随机抽样结果，因为批量 multinomial 会改变 RNG 消耗顺序；
- 但必须保证相同概率分布、相同无放回规则、相同 k 和相同 fallback 语义；
- 如果项目要求严格复现实验，保留旧选择路径作为 legacy 模式；
- 增加 adaptive_position_selection_impl：legacy 或 grouped；
- 默认先使用 grouped，但 use_adaptive_position_aug=False 不受影响。

如果 grouped 实现风险过高，允许保留 batch 级循环，但绝对不能保留“batch × position × neighbors”的多层循环。

十三、性能验收标准
==================================================

请用相同机器、相同数据集、相同 seed、相同 batch size、相同配置进行比较。

已知参考：
- 原始 KGSCL：约 134～140 秒/epoch；
- 优化前自适应版本：约 576～581 秒/epoch。

要求输出：

1. 优化前每阶段耗时；
2. 优化后每阶段耗时；
3. reliability 预计算时间；
4. 单 epoch 总时间；
5. 相对原始 KGSCL 的时间倍率；

7. GPU 显存变化；
8. CPU 内存变化；
9. original forward 次数；
10. fallback ratio 是否一致。

目标：
- 优先将自适应版本降低到 200 秒/epoch以内；
- 理想目标为不超过原始 KGSCL 的 1.3～1.5 倍；
- 若未达到目标，必须继续报告剩余最大瓶颈；
- 不允许通过改变功能达到时间目标。

性能测量要求：

2. 排除第一个 CUDA warm-up batch；
3. 至少统计 3 个完整 epoch 的平均值；
4. 关闭 profiler 后再测最终时间；
5. 保持日志级别一致；
6. 不同时运行其他任务。

==================================================
十四、建议的实现顺序
==================================================

请按以下顺序修改，每一步都做计时：

步骤 1：确认没有额外 original forward。
步骤 2：预计算 sub_reliability_table 和 ins_reliability_table。
步骤 3：训练时改成 table[item_seq] 索引。
步骤 4：向量化 masked min-max normalization。
步骤 5：向量化 position importance。
步骤 6：向量化 position scores。
步骤 7：优化统计量累计，减少 .item() 和同步。
