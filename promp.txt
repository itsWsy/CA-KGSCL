你现在需要在当前 KGSCL 代码库中修改已经实现的 V1：Multi-Positive View-Target Contrastive Learning。

当前问题：
现有 MP-VT 使用如下正样本集合：

P_t = {raw target item} ∪ TopM(N_S(raw target item))

并且所有正样本在分子中权重相同。实验中 mp_vt_top_m=3 和 mp_vt_top_m=5 都没有超过原始 KGSCL，说明额外 substitute 正样本噪声较大，且 raw target 被重复用于 V2T 可能削弱 KGSCL 原始 target substitution 的作用。

现在需要将 MP-VT 修改为：

Weighted MP-VT with Target Substitution

核心目标：

1. 保留 KGSCL 原始 target substitution 的设计；
2. 不再默认把 raw target item 放入 V2T 正样本集合；
3. 将原始 KGSCL 生成的 target_sub_item 作为主正样本，权重为 1.0；
4. 将 raw target 的 TopM substitute neighbors 作为辅助正样本，权重为 rho，默认 rho=0.1；
5. 辅助正样本只作为弱语义监督，不与主正样本同等对待；
6. 当没有额外 substitute neighbors 时，新的 weighted MP-VT 应退化为原始 KGSCL 的单正样本 target-substitution V2T；
7. 不要实现假负样本过滤，不要实现自适应增强，不要实现置信度邻居采样。

)

输入：

* targets: raw target item tensor, shape [B]
* target_sub_items: KGSCL 原始 target substitution 得到的主正样本 tensor, shape [B]
* sub_neighbors: substitute relation set
* corr_score: co-occurrence/correlation score
* top_m: 额外 substitute 正样本数量
* num_items: item 总数
* pad_id: padding id，通常为 0
* use_raw_target: 是否将 raw target 作为辅助正样本，默认 False

输出：

* extra_pos_sets: List[List[int]]
  每个样本对应一个辅助正样本列表，不包含主正样本 target_sub_item。

构造逻辑：
对 batch 中每个样本：

1. 取 raw target t；
2. 取主正样本 target_sub ts；
3. 从 substitute relation set 中获取 t 的 substitute neighbors；
4. 过滤掉：

   * pad_id；
   * 非法 item id；
   * target_sub_item；
   * raw target item，除非 use_raw_target=True；
   * 重复 item；
5. 如果 corr_score 可用，则按照 get_corr_score(t, neighbor) 从高到低排序；
6. 取前 top_m 个作为辅助正样本；
7. 如果没有可用 neighbor，则该样本 extra_pos_set 为空列表。

注意：

1. target_sub_item 不应出现在 extra_pos_sets 中，因为它已经作为主正样本；
2. raw target 默认不应出现在 extra_pos_sets 中；
3. 不能把 complementary / also_buy 关系用于 View-Target 的额外正样本，只能使用 substitute / also_see 关系；
4. item id 如果是 1-based，必须保证 0 不进入正样本；
5. Embedding 大小如果是 num_items + 1，合法 id 通常为 1 到 num_items。

请实现或复用以下辅助函数：

* get_substitute_neighbors(item_id, sub_neighbors)
* get_corr_score(src, dst, corr_score)

它们需要兼容 dict、list、numpy array、torch tensor、scipy sparse matrix 等当前项目中可能已有的数据结构。

五、实现 Weighted MP-VT Loss
新增函数：

weighted_multi_positive_view_target_loss(
h_view,
main_pos_items,
extra_pos_sets,
item_embedding,
tau=1.0,
extra_weight=0.1
)

输入：

* h_view: torch.Tensor, shape [B, D]
* main_pos_items: torch.LongTensor, shape [B]，这里必须是 target_sub_item
* extra_pos_sets: List[List[int]]
* item_embedding: torch.Tensor, shape [num_items, D]
* tau: float
* extra_weight: float，辅助正样本权重 rho

输出：

* loss: scalar tensor

损失形式：

L = - log [
exp(sim(h, e_main) / tau)
+
rho * sum_{v in extra_pos} exp(sim(h, e_v) / tau)
------------------------------------------------------
sum_{v in V} exp(sim(h, e_v) / tau)
]

要求使用 logsumexp 的数值稳定形式，不要直接 exp 后求和。

实现逻辑：

1. logits = h_view @ item_embedding.T / tau，形状 [B, num_items]；
2. 构造 pos_weight，形状 [B, num_items]，初始为 0；
3. 对每个样本 b：

   * pos_weight[b, main_pos_item] = 1.0；
   * 对 extra_pos_sets[b] 中的 item，pos_weight[b, item] = extra_weight；
   * 如果 extra item 与 main item 相同，不重复；
4. weighted_pos_logits = logits + log(pos_weight + eps)；
5. 对 pos_weight <= 0 的位置 mask 为 neg_inf；
6. log_pos = torch.logsumexp(weighted_pos_logits, dim=1)；
7. log_all = torch.logsumexp(logits, dim=1)；
8. loss = -(log_pos - log_all).mean()。

参考实现：

def weighted_multi_positive_view_target_loss(
h_view,
main_pos_items,
extra_pos_sets,
item_embedding,
tau=1.0,
extra_weight=0.1
):
logits = torch.matmul(h_view, item_embedding.transpose(0, 1)) / tau
batch_size, num_items = logits.size()
device = logits.device

```
pos_weight = torch.zeros_like(logits)

main_list = main_pos_items.detach().cpu().tolist()

for b in range(batch_size):
    main_item = int(main_list[b])
    if 0 <= main_item < num_items:
        pos_weight[b, main_item] = 1.0

    for item in extra_pos_sets[b]:
        item = int(item)
        if 0 <= item < num_items and item != main_item:
            pos_weight[b, item] = extra_weight

neg_inf = torch.finfo(logits.dtype).min
weighted_pos_logits = logits + torch.log(pos_weight + 1e-12)
weighted_pos_logits = weighted_pos_logits.masked_fill(pos_weight <= 0, neg_inf)

log_pos = torch.logsumexp(weighted_pos_logits, dim=1)
log_all = torch.logsumexp(logits, dim=1)

loss = -(log_pos - log_all).mean()
return loss
```

注意：

1. 如果原始 KGSCL 的 similarity 使用 dot product，则保持 dot product；
2. 如果原始 KGSCL 对 embedding 做了 normalize，这里也保持一致；如果原始没有 normalize，不要额外加；
3. h_view、main_pos_items、item_embedding 必须在同一 device；
4. extra_pos_sets 是 Python list 没问题，但 item id 必须合法；
5. 当 extra_pos_sets 全为空时，loss 应该等价于单正样本 target_sub_item 的 View-Target CL。
